# Agent CLI 选择指南

## 设计原则

Hermes 用便宜/免费模型做调度，四步法是能力升级器。每个步骤选择 Agent 时考虑：
1. **能力加强**：哪个 Agent 在这一步能力最强
2. **性价比**：花最少钱补齐 Hermes 的短板

## 按步骤推荐

### Step 1 审查（需求分析 + 代码理解 + 结构化输出）

Hermes 做不好这步的原因：不能可靠理解人类混乱的表达、不能系统性分析代码。

| 推荐级别 | Agent           | 模式      | 理由                                    |
|---------|----------------|-----------|----------------------------------------|
| 能力加强 | Kimi K3        | `kimi -p` | K3 推理能力强，中文理解好                |
| 能力加强 | Claude Code    | `claude -p` | 理解能力最强（需订阅）                 |
| 性价比   | MiMo Code      | `mimo run` | 免费，中文好，能读代码+结构化输出        |
| 性价比   | Codex CLI      | `codex exec` | 已有订阅，够用                        |

### Step 2 方案（逻辑推理 + 生成修复计划）

Hermes 做不好这步的原因：不能可靠地生成带文件/行号/代码的修复方案。

| 推荐级别 | Agent           | 模式             | 理由                                    |
|---------|----------------|------------------|----------------------------------------|
| 能力加强 | Kimi K3        | `kimi --plan`    | 专用方案模式，结构化输出                 |
| 能力加强 | Claude Code    | `claude -p`      | 方案质量最高（需订阅）                  |
| 性价比   | MiMo Code      | `mimo run`       | 免费，能生成合理方案                    |
| 性价比   | Codex CLI      | `codex exec`     | 已有订阅，方案质量稳定                  |

### Step 3 执行（按方案修改 + 验证）

Hermes 能做这步的原因：有人告诉它改什么，它按方案执行就行。

| 推荐级别 | Agent           | 模式             | 理由                                    |
|---------|----------------|------------------|----------------------------------------|
| 能力加强 | Aider          | `aider --yes`    | 编辑最强，lint/test/git 集成            |
| 能力加强 | Claude Code    | `claude -p`      | 权限控制好，自带 git（需订阅）          |
| 性价比   | MiMo Code      | `mimo run`       | 免费，能改+能验证，比 Hermes 强         |
| 最便宜   | Hermes patch   | `patch()`        | 免费，但不能验证、不能理解逻辑          |

**关键洞察：** Step 3 的难度 = 理解要改什么（需要强模型）+ 执行修改（工具就行）。当 Step 2 已经给出详细方案时，Hermes patch 就够用。当需要"改完还要验证+理解报错再修复"时，升级到 MiMo Code。

### Step 4 复审（独立视角 + 验证结果）

Hermes 做不好这步的原因：不能独立验证、可能重复自己的盲区。

| 推荐级别 | Agent           | 模式             | 理由                                    |
|---------|----------------|------------------|----------------------------------------|
| 能力加强 | Gemini CLI     | `gemini`         | 不同模型族，避免盲区重复                 |
| 能力加强 | Claude Code    | `claude -p`      | 审查质量最高（需订阅）                  |
| 性价比   | Codex CLI      | `codex exec`     | 已有订阅，独立 session                  |
| 性价比   | MiMo Code      | `mimo run`       | 免费，但同模型族可能重复盲区            |

**关键洞察：** Step 4 应用不同模型/提供商。同模型族在不同 session 中可能重复盲区。

## 推荐组合

### 省钱模式（几乎免费）
```
Step 1: MiMo Code（免费）
Step 2: MiMo Code（免费）
Step 3: Hermes patch（免费）
Step 4: Codex CLI（已有订阅）
```

### 加强模式（质量最高）
```
Step 1: Kimi K3（推理最强）
Step 2: Kimi K3 --plan（专用方案模式）
Step 3: MiMo Code（免费+够用）
Step 4: Gemini CLI（不同模型族）
```

### 均衡模式（当前方案）
```
Step 1/2/4: Codex CLI（已有订阅）
Step 3: Hermes patch
```

### 独立复审模式（推荐搭配）
```
Step 1/2: Codex CLI
Step 3: MiMo Code
Step 4: Gemini CLI（不同模型族）
```
